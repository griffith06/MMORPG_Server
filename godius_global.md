# ** 가디우스 글로벌/Steam 버전 리마스터링 버전 **

## 📋 프로젝트 개요

### 핵심 목표
- ✅ 게임성 유지
- 🎨 그래픽 및 해상도 현대화 리마스터링
- ⚡ 서버 성능 50% 이상 개선
- 🌐 Steam 기반 글로벌 서비스 가능
- 💰 운영 비용 및 리스크 감소

## 1️⃣ 프로젝트 배경 및 목적

### 프로젝트 필요성
- 기존 가디우스의 검증된 게임성 유지하면서 현대화
- 현재 PC 환경 및 글로벌 서비스 환경에 맞게 기술적 업그레이드
- Steam 플랫폼을 통한 글로벌 시장 진출
- 운영 비용 절감 및 향후 확장성 확보

---

## 2️⃣ 현재 상황 분석

### 클라이언트 현황
| 항목 | 현재 상태 | 문제점 |
|------|-----------|--------|
| 해상도 | 800×600 고정 | 고해상도/와이드 미지원 |
| 컬러 | 256컬러 (8bit) | 현대적 그래픽 품질 한계 |
| 렌더링 | CPU 기반 | 성능 병목, 밀집 시 저하 |
| 플랫폼 | 독립 클라이언트 | Steam 생태계 미연동 |

### 서버 현황
- ✅ **장점**: 국내 서비스 안정적 운영 중
- ⚠️ **문제점**:
  - 동시접속 대비 서버 자원 사용 효율 낮음
  - 서버 성능 병목으로 서버 수 과다 사용
  - 운영 비용 증가

## 3️⃣ 프로젝트 목표

### 클라이언트 목표
1. **해상도 현대화**: FHD/QHD/4K 지원, 와이드 스크린 대응
2. **그래픽 품질 향상**: TrueColor 전환, GPU 렌더링
3. **성능 개선**: 플레이어 밀집 시 안정적 프레임 유지
4. **Steam 연동**: Steamworks SDK 통합

### 서버 목표
1. **성능 향상**: 서버 처리 성능 50% 이상 개선
2. **효율 개선**: 동일 동접 기준 서버 자원 사용량 절감
3. **안정성 유지**: 기존 검증된 안정성 유지
4. **확장성 확보**: 글로벌 서비스 확장 대비

## 4️⃣ 클라이언트 개발 범위

### 4-1. 렌더링 구조 현대화
**현재 → 목표**
- DirectDraw/GDI → DirectX GPU 렌더링
- CPU BitBlt → 배치 렌더링

**기대 효과**
- 플레이어 밀집 시 성능 저하 해결
- 안정적인 프레임 유지

### 4-2. 해상도 확장
**기술 방식**
- 내부 논리 해상도: 800×600 유지 (UI 호환성)
- 출력 해상도: FHD/QHD/4K 지원
- 와이드 스크린 대응

### 4-3. TrueColor 전환
**작업 내용**
- 256컬러 → TrueColor 변환
- 투명 색상(Color Key) 유지
- ARGB1555 포맷 사용 (상황에 따라 DXT)

**작업 방식**
- 자동 변환 툴 제작
- 기존 그래픽 표현 유지

### 4-4. 리소스 리마스터링
**방법론**
- Stable Diffusion 기반 업스케일
- 기존 감성 및 스타일 유지 중심
- 배경(Tile) 및 캐릭터 리소스 개선

### 4-5. Steam SDK 연동
**기본 연동 범위**
- Steamworks SDK 통합
- Steam 계정 기반 로그인
- Steam Overlay 지원
- Steam 전용 빌드 구성

**추가 협의 필요** (선택사항)
- 업적 시스템
- 친구 기능
- 상점 연동

### 4-6. 다국어 지원
**구조 설계**
- 문자열 리소스 분리
- 다국어 로딩 구조
- 언어별 리소스 관리

**지원 언어**
- 1차: 영어 (필수)
- 2차: 확장 가능 구조 확보

- 서버 역할 중복 최소화
